Get started

Hello and welcome! DC++ is a free, open-source client for Direct Connect file sharing networks. As it is fully compatible with and supports connection to hubs using the earlier NeoModus Direct Connect (NMDC) protocol, now its primary purpose is to be a client for a new generation of file sharing using a protocol called Advanced Direct Connect (ADC) .

The newest version of DC++ should work well on any edition of modern Windows NT based operating systems such as Windows 2000, XP and Vista. Some older versions will run on Win9x/ME but using DC++ on these ancient operating systems is neither recommended nor supported. As long as it downloaded from the trusted home page , DC++ contains no viruses or adware of any kind. DC++ is for free so if you paid for it, ask for refund!!

Structure of the Direct Connect network

The Direct Connect (DC) network is a decentralized network, made up of individual servers (hubs) that users join to share files with other members on that hub. Hubs feature a list of users connected to them. Users can search for files and download them from other connected users. A hub only helps to find files and connect users, it does not store any files so file transfers are done directly between clients, in peer-to-peer fashion.

Every hub acts as an instant messaging server as well so users can chat with other users of the same hub by either using the main chat (visible for every user of a hub) or starting private conversations with a particular user. Each hub is individually run by a fellow user of the Direct Connect network, and may have certain themes to the content on the hub. Hub owners set up rules to regulate their hub as they see fit. Many have requirements on the total size of the files that their members share (share size), and restrictions on the content and quality of shares. A hub may choose certain individuals called operators who are responsible to help normal users and and force them to observe the rules of the hub.

There are two kinds of hubs exist: Public hubs which are available for everyone (who meet the rules of a certain hub) and require no registration for connect (they may need registration to be able to use their additional features though). On the other hand Private hubs always require user authentication at login. Authentication is done by a registered username and password combination. So one needs an invitation from the owner or an existing member of a private hub before beeing able to connect.

There are Hublists hosted by individual hublist servers to provide lists of available hubs. These lists contain the name, address and description of each hub along with other data. Some big hublists try to contain almost all available hubs while others contain only a list of hubs for a special interest.

Connecting to DC hubs using DC++

DC++ has a few public hublist servers configured by default. They are the most popular and reliable ones so they are available most of the time. You can open up the Public Hublist window by using View/Public Hubs menuitem or clicking the appropriate icon in the Toolbar. You are able to add other hublist servers or manage a list of your own Favorite hubs. If you simply want to connect to a particular hub with a known address you can do it by using the File/Quick Connect menu command.
However before connecting to any hub you are recommended to be familiar with the basic principles of the DC network and have your client set up correctly to enjoy all the possibilities and benefits of Direct Connect.

Checking software firewall settings

Software firewalls are often cause problems as they can completely prevent DC++ from being connected to Internet. You can skip this section if you can download hublists and connect to hubs. However if you can't connect to any hub or even can't download any hublist while your other programs able to access the Internet then its possible that a software firewall blocks DC++.

Newer Windows operating systems (XP with Service Pack 1,2 or 3 and Vista) have a built in firewall enabled by default. Moreover, almost all anti-virus suites include firewall component (like Norton, McAfee, CA, Eset, Kaspersky, F-Secure, etc..) with their "typical" install method. So you can still have a firewall running even if you've never installed a separate firewall product or never seen a slight sign of its operation.

The easiest way to check if DC++ is blocked to go the Help/About menu in DC++ and check if it can download the Latest Version number (bottom box of the window). A version number should appear in the box soon. If you get error message instead it means something keeps DC++ from accessing the Internet.

The way of unblocking programs in firewalls varies product by product. Check your firewall's help or manual how to do this. The best practice is to exit DC++ and remove ALL the rules for DC++ in the firewall configuration. In most cases the time of next start of DC++ you'll be shown a popup window where you can re-enable DC++. You can find configuration method for some popular firewall software following this link.

Setting up your connection

To make your life easier while using DC++ you should find out what connection mode suits you best. There are two connection methods availabe in DC++, active mode and passive mode. While choosing passive mode is easier it is not efficient (neither for the user nor for the network). Active mode is always preferred, here's the explanation why is it better for you : Why do I need active mode?

It is very important to decide what active connection mode available for you. It depends on your network stucture so it is required to know how do you access the Internet. Is it a direct connection through a modem or you have a local network and you access the Internet through a router or other gateway? If you unsure here is a FAQ for help to decide .

If you're still unsure what network devices you use to connect to the Internet then use Google ! You got a very helpful information on every single network device : the Model Number, look at your device(s), find the Model Number and a search for it will tell you what your device actually is. Don't skip this step since many ISPs call their device as a simple modem while it is actually a router.

So to choose a proper active connection mode the most important thing is to know : are you behind a router or a gateway or do you access the Internet directly. There is another way to decide : check your external and internal IP addresses. If these IP adresses do NOT match then you access the Internet through a local gateway or a router.

  • To see your internal IP address go to the Windows Start menu, click Run... and type : cmd /k ipconfig
  • To see your external IP address go to one of the following websites : www.whatismyip.com , www.dslreports.com/ip , www.ip2location.com

If you are still unsure you may want to consult your ISP support hotline, a system administrator (if you are in an office), etc...

When you look at the Connection Settings you will find 3 options for active mode and the last option for beeing passive as a last resort. For beeing active

  • You can use Direct Connection option ONLY if you don't have a router/gateway.
  • If you have a router/gateway you can use ONLY the 2nd or the 3rd option : Firewall with manual port forwarding or Firewall with UPnP
For setting up any of the three active mode options follow the active mode setup FAQ.

If none of the above helped or you are not able to use active mode in your network (eg. you have no access to the configuration page of the router/gateway or you have firewall service provided by your ISP,...) then must choose Firewall (passive, last resort) option in Connection Settings. Of course choosing this option means that you should live with all the disadvantages that passive mode has...

Help resources

Need more help?

  • You can find FAQs and explanation of ALL settings in the built-in context sensitive Help of DC++. Just move the question mark icon from the Toolbar to any control you unsure about or just select a control and press F1 key.
  • There is a complete Direct Connect FAQ maintained at dslreports.com.
  • There are more answers to the most frequent support problems at the FAQ section of the DC++ Launchpad page.
There is an online support forum and a bug & feature tracker also available. You can access these pages through the Help menu of DC++ or from the links page. Please go through all the documentation mentioned above before asking for support. The best part of the questions asked in the support page are already answered in the FAQs or even in the built-in Help.
Important : before asking for support please keep in mind that DC++ is a client for connecting to a network and developers of DC++ have no connection to the owners of individual hubs or hublist servers. So when you have any problem with a particular hub or hublist server (and you can use others without problems) DC++ support team is unable to help you in any way.
