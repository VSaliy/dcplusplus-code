User matching definition

This dialog configures one user matching definition. More information about user matching definitions is available in the page about the User matching settings page.

Name

The name this user matching definition will be identified as in the User matching and the Styles settings page.

Match favorite users

If checked, users will have to have been marked as favorite users from the Users window in order for this user matching definition to be applied to them.

Match operators

If checked, users will have to be hub operators in order for this user matching definition to be applied to them.

Match bots

If checked, users will have to be hub bots in order for this user matching definition to be applied to them.

Rules

Add a rule

Add a line to define a rule that users will have to match in order for this user matching definition to be applied to them.

Field selection drop-down (left)

The user property to be matched against for the current rule.

Pattern text-box

Pattern for the current rule.

Method selection drop-down (right)

Define how the pattern will be matched against the user property selected in the field selection drop-down.

X button (right)

Remove the current rule.

Always show chat messages

Chat messages for users that match this user matching definition will always be shown. Use this to override other "chat ignore" definitions.

Ignore chat messages

Do not show chat messages for users that match this user matching definition.
