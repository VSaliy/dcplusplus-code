Search

The Search window make possible searching for files in shares of the users of the connected hubs. You can provide one or more keywords and optionally set some search filters described below and press the Search button to start a search. DC++ will send out the search request to the hub(s) you are connected and they will pass the request to all clients.

When a client matches a serach request it'll send the results to the requester. The results will be displayed in the search results list of the Search window.

You can open several Search windows and you can define search keywords and set filters independently so you are able to start multiple searches simultaneously. However search requests are sent out only once, when the Search button pressed. After that - until the button pressed again - a Search window only listens for and displays search results. (It means that those clients who logged in after you pressed the Search button won't send you search results until you repeat the search.)

You may not start or repeat searches as soon as you like. There is a few seconds of minimum interval between two searches defined in DC++. You will get a countdown message in the Status bar if you try to search sooner than allowed and you are able to repeat the search only when the client is ready for search again. Hubs may also limit searches by checking for forbidden keywords or blocking repeated searches for the same terms in a certain amount of time. Some hubs have predefined minimum search intervals for all searches or they even totally block searches if you are in passive mode.

Note that if you search on NMDC hubs, results are transferred to all open Search windows. The windows then display only those results that match their search criteria. So when a similar search sent out from a new Search window, its normal that some hits displayed in some other Search windows opened before. The number of hits filtered out are shown in the status bar of every window. In case of searches on ADC hubs every Search window will pick its own search results only.

Search hits with TTH are always matched to the Download queue and the new sources are automatically added to the queue items unless they are removed or marked as "bad" sources before. The filelists of all newly discovered sources will be automatically matched for possible other queued files if Automatically match queue for search hits setting is enabled. This can generate lots of filelist traffic when large amount of new sources are found for queued files.

If you often get too few search results or no results at all, refer to the Search troubleshooting FAQ.

Search Options

Search for
This is the set of search strings to search for. Words beginning with "-" will be filtered away.
Purge
Clear the contents of the search history.
Search
Start searching for the set of search strings defined above. Note; you may not start too many searches in a small amount of time.
Size
Filter away files depending on their file size.
File type
Filter away files depending on their file type (extension). Not all the possible extensions will be filtered out in certain file type groups. If you search for newer or obscure file types then is recommended to use the Any option and include the file extension to the search string instead. Options other than Any, Directory & TTH are ignored for searches on ADC hubs.
Only users with free slots
Filter away files depending on the user's free slots.
Hide files already in share
Filter away files that are already in share.
Hubs
Search only selected hubs.

Search results columns

File
The file name.
User
The user's nick name.
Type
The file's extension (file type).
Size
The file's size.
Slots
x/y: x slots available of a total y.
Path
The path in the user's file list where the file is stored.
Connection
The user's possible line speed in upload.
Hub
In which hub the user is in.
Exact size
The exact size of the file.
IP
The user's IP.
TTH root
The TTH root for the file.
CID
The unique ID that DC++ use to identify other users.

Status bar

Unnamed checkbox
Toggle the displayment of the search options.
# items
The amount of items that are displayed in the window.
# filtered
The amount of files that have been filtered away from the view.
